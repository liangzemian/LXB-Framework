# 通知触发任务

通知触发任务用于“收到某类通知后自动执行任务”。AutoLXB 会读取手机通知，按你配置的规则进行匹配；规则命中后，再执行对应的自动化任务。

适合场景：

- 收到某个 App 的通知后自动处理。
- 收到特定联系人、群聊或关键词消息后自动回复。
- 收到订单、物流、工单等通知后进入 App 查看详情。

## 新建规则

入口：`任务页 -> 自动化 -> 通知触发任务 -> 新建`。

新建或编辑通知触发任务时，前端会按卡片分成几块：

1. **基本信息**：填写规则名称、触发后执行的任务描述和任务操作文档。
2. **触发条件**：选择监听 App，填写标题 / 正文匹配、触发间隔和触发时段，必要时启用 LLM 条件。
3. **执行偏好**：设置是否按任务路线执行、是否录制触发后的任务屏幕。
4. **保存触发器**：点击保存 / 提交后，规则才会出现在通知触发任务列表里。

## 字段说明

下面的字段名按前端实际显示整理。

| 前端字段 | 说明 | 是否必填 | 示例 |
| --- | --- | --- | --- |
| 规则名称 | 给这条规则起一个方便识别的名字。 | 否 | 回复微信消息 |
| 任务描述（触发后执行什么） | 规则命中后要执行的任务。 | 是 | 打开微信，找到“WuWei”，回复他的最新消息一次 |
| 任务操作文档（可选） | 给模型看的补充步骤，用来说明具体怎么执行。 | 否 | 打开微信后，点击右上角搜索，输入“WuWei”，找到聊天后回复 |
| Package 匹配（必填）- 选择应用 | 选择要监听哪个 App 的通知。当前界面从本地 App 快照中选择一个包名。 | 是 | 微信（com.tencent.mm） |
| 标题匹配（可选） | 匹配通知标题。留空表示不限制标题。 | 否 | WuWei |
| 正文匹配（可选） | 匹配通知正文。留空表示不限制正文。 | 否 | 在吗 |
| 触发间隔（秒） | 同一规则两次触发之间的最短间隔，避免短时间内重复执行。 | 是 | 60 |
| 触发时段开始（HH:mm，可选） | 规则只在某个时段内生效时填写开始时间。 | 否 | 12:00 |
| 触发时段结束（HH:mm，可选） | 规则只在某个时段内生效时填写结束时间。 | 否 | 13:00 |
| 启用 LLM 条件判断 | 开启后，模型会在普通匹配通过后再判断通知是否符合自然语言条件。 | 是 | 开 / 关 |
| LLM 条件（可选） | 给模型看的判断标准。只有开启 LLM 条件判断后才显示。 | 否 | 消息来自 WuWei，且是在询问我是否有空 |
| 按路线执行 | 前端显示为“任务路线模式”，可选关 / 开。开启后，命中规则后执行任务时会优先使用保存过的路线。 | 是 | 开 |
| 打开任务路线编辑页 | 只在已经保存过的通知触发任务里显示，用于进入该任务自己的路线编辑页。 | 否 | - |
| 导出便携任务 | 只在已经保存过的通知触发任务里显示，用于导出任务配置和任务路线。 | 否 | - |
| 为通知触发任务录屏 | 开启后会录制规则命中后的任务执行过程，方便排查问题。 | 是 | 开 |

!!! note "开始或结束留空表示任意时段"
    前端提示“开始或结束留空时，表示任意时段都可触发”。如果不需要限制触发时间，两个时段字段都可以留空。

## “标题”和“正文”是什么？

下面是一条微博通知示例：

![通知示例](../assets/images/notify_example.jpg)

在这条通知中：

- “极地邮轮暴发致命病毒致 3 死”是标题。
- “载 220 人邮轮……”是正文。

标题匹配和正文匹配都是 **包含匹配**。也就是说：

- 标题匹配填 `极地邮轮`，可以匹配到标题中包含“极地邮轮”的通知。
- 正文匹配填 `220 人`，可以匹配到正文中包含“220 人”的通知。
- 如果标题和正文都填写，则两者都需要匹配成功。
- 如果某一项留空，就表示这一项不限制。

!!! tip "标题和正文不用都填"
    如果只关心谁发来的通知，可以只填标题；如果只关心消息内容，可以只填正文。两个都填会更精确，但也更容易因为通知文案变化而匹配不到。

## LLM 条件什么时候用？

普通关键词适合明确、稳定的规则，例如“标题包含某个人名”“正文包含某个订单号”。

如果规则比较模糊，可以启用 LLM 条件判断，例如：

```text
这条通知是在询问我是否有空，或者要求我尽快回复。
```

LLM 条件会在普通包名、标题、正文匹配通过后再执行。它可以提高判断灵活性，但也会增加模型调用成本和延迟。

## 使用建议

- 先用 Package 匹配 + 一个标题或正文关键词做简单规则，确认能触发后再加复杂条件。
- 触发间隔不要设得太短，避免同一批通知连续触发多次任务。
- 如果任务会自动回复消息，建议先用安全文本测试。
- 如果通知文案经常变化，少填标题 / 正文，多使用 LLM 条件描述意图。
- 如果任务执行路径固定，跑通后可以保存路线并开启“按路线执行”。

## 导入任务

导入的通知触发任务默认是停用状态。导入后建议先检查 Package 匹配、标题 / 正文匹配、LLM 条件、任务描述和路线设置，确认无误后再启用。
