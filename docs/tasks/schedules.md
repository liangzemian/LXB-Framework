# 定时任务

定时任务用于让手机在指定时间自动执行任务，适合每日签到、定时查看订单状态、定时发送提醒等固定时间或固定周期的自动化场景。

## 创建定时任务

入口：`任务页 -> 自动化 -> 定时任务 -> 新建`。

新建或编辑定时任务时，前端会按卡片分成几块：

1. **基本信息**：填写任务名称、任务描述和用户操作文档。
2. **执行目标**：选择运行日期、时间、重复方式和优先打开的 App。
3. **执行偏好**：设置是否按任务路线执行、是否录制任务屏幕。
4. **保存定时任务**：点击保存 / 提交后，任务才会出现在列表里。

## 字段说明

下面的字段名按前端实际显示整理。

| 前端字段 | 说明 | 是否必填 | 示例 |
| --- | --- | --- | --- |
| 名称（可选） | 给这个定时任务起一个方便识别的名字。留空也可以保存。 | 否 | 点生椰拿铁 |
| 任务描述（必填） | 告诉 AutoLXB 要完成什么任务。建议写清目标 App、目标页面和最终动作。 | 是 | 打开瑞幸咖啡，帮我点一杯生椰拿铁 |
| 用户操作文档（可选） | 给模型看的补充步骤。适合写页面顺序、按钮名称、特殊要求。 | 否 | 进入瑞幸咖啡后，点击菜单，搜索生椰拿铁，选择热饮、大杯、堂食 |
| 运行时间 | 当前选择的触发时间预览。由下面的“选择日期”和“选择时间”决定。 | 是 | 2026-05-06 11:00 |
| 选择日期 | 打开日期选择器，设置任务首次或单次执行日期。 | 是 | 2026 年 5 月 6 日 |
| 选择时间 | 打开时间选择器，设置任务触发时间，使用 24 小时制。 | 是 | 11:00 |
| 重复 | 选择任务执行方式：单次 / 每天 / 每周。 | 是 | 每天 |
| 已选日期 | 只有选择“每周”时显示，用于选择周一到周日哪些天执行。 | 每周时必选至少一天 | 周一、周三、周五 |
| 包名（从本地快照选择） | 选择任务优先打开的 App。能选就尽量选，可以减少模型判断目标 App 的不确定性。 | 否 | 瑞幸咖啡（com.lucky.luckyclient） |
| 按路线执行 | 前端显示为“任务路线模式”，可选关 / 开。开启后，如果该任务已保存路线，会优先回放路线。 | 是 | 开 |
| 打开任务路线编辑页 | 只在已经保存过的定时任务里显示，用于进入该任务自己的路线编辑页。 | 否 | - |
| 导出便携任务 | 只在已经保存过的定时任务里显示，用于导出任务配置和任务路线。 | 否 | - |
| 录制任务屏幕 | 开启后会把任务执行过程保存到 `Movies/lxb`，方便失败后排查。 | 是 | 开 |

!!! note "新建时看不到路线编辑和导出"
    “打开任务路线编辑页”和“导出便携任务”只有任务配置至少保存一次后才会显示。新建任务时先填写并提交，之后再回来编辑路线或导出。

## 任务描述和用户操作文档怎么分工？

**任务描述（必填）** 写最终目标，尽量简短明确：

```text
打开瑞幸咖啡，帮我点一杯生椰拿铁
```

**用户操作文档（可选）** 写执行细节，适合补充模型可能不知道的页面路径：

```text
进入瑞幸咖啡后，点击菜单，搜索生椰拿铁。选择大杯、热饮、堂食。如果出现加购或推荐弹窗，关闭后继续下单。
```

## 使用建议

- 第一次创建定时任务前，建议先用快速任务跑一遍同样的任务描述。
- 如果任务会付款、下单或发送重要消息，先用低风险内容测试。
- 如果任务路径稳定，跑通后可以保存路线，再开启“按路线执行”。
- 修改时间或重复方式不会影响已经保存的路线；你可以放心调整执行时间。

## 导入任务

导入的定时任务默认是停用状态。导入后建议先检查任务描述、包名、执行时间、重复方式和路线设置，确认无误后再启用。
